4.2. Version control

  1. Git

    Some Important steps

    1. Before first commit to check the repo status
      git status
    2. Add files to staging area
      • To add a particular file in the staging area
      git add <file_name>
      • To add all files to the staging area
      git add -A
    3. Remove files from staging area
      git reset <file_name>

      To remove all files from the staging area

      • To put all files in the staging area to the repository we have to perform a commit
      git commit -m "My first commit"
    4. To view logs
      git log
    5. Pull and push changes to remote repository
      git pull origin main

      git push origin main
